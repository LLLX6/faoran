# تقرير إصدار خدماتي v58

## الهدف

تثبيت رحلة الطلب والمزود والإدارة قبل الإطلاق، وإصلاح الأعطال المرئية في الهاتف من دون إعادة بناء المشروع أو تغيير بنية البيانات الحالية.

## التنفيذ

- أضيف عارض صور أعمال كامل داخل التطبيق، مع تنقل وإغلاق يعيد المستخدم إلى بطاقة المزود المفتوحة.
- أضيفت معاينة تقارير داخلية قابلة للإغلاق والطباعة PDF.
- عولجت إشعارات النظام المولدة بالعربية عند استخدام الواجهة الإنجليزية.
- أضيف مسار استرجاع آمن: طلب مراجعة، إصدار رمز مؤقت لمرة واحدة من الإدارة، وعبارة واتساب جاهزة. لا يُكشف رمز الدخول الأصلي.
- عولج دخول المزود عند وجود حساب مستخدم وسجل مزود أو أكثر للهاتف نفسه باختيار السجل الذي يطابق الرمز فعلًا.
- دُققت مطابقة الطلبات ورسائل الإرسال، وأصبح التطبيق يميز بين المزود المطابق والمزود الذي وصله الإشعار فورًا.
- ثُبت وصول طلب ترقية المزود إلى الإدارة وانعكاس تفعيل الباقة على حساب المزود وصلاحياته.
- حسّن محرر الصور التصغير والخلفية المشتقة من الصورة، وصُغرت بطاقات الإرشاد.
- عولج رأس المزود والإدارة على 320px، وثُبت اسم المزود وشعاره وأزراره داخل الشاشة.

## نتائج التحقق

- Python compile: ناجح.
- Unit tests: 14 من 14 ناجحة.
- API smoke: ناجح، ويشمل الطلبات والمطابقة والترقية والاسترجاع.
- Security API: ناجح في تقليل البيانات وCORS والوسائط الخاصة وIDOR والجلسات والقفل وحدود الطلبات والكاش الخاص.
- UI smoke: ناجح على 320x740 و390x844 و1280x900.
- الأداء المحلي: First Contentful Paint حوالي 444ms، والجاهزية التفاعلية حوالي 1165ms.
- تدقيق الحزمة: لا توجد ثغرات معروفة في `pywebpush==2.3.0` وقت الفحص.

## إعداد خارجي

إرسال واتساب أو SMS تلقائيًا يحتاج مزود رسائل معتمد ومفاتيح خادمية في متغيرات البيئة. من دون ذلك تعمل الإدارة بإصدار رمز مؤقت وفتح رسالة واتساب جاهزة يدويًا، ولا يعرض التطبيق نجاح إرسال وهميًا.
