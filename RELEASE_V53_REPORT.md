# تقرير إصدار خدماتي v53

تاريخ الإعداد: 2026-07-22

## هدف الإصدار

تحسين الإصدار القائم دون إعادة بناء التطبيق أو كسر بياناته، وجعل رحلة المستخدم والمزود والإدارة أكثر وضوحًا على الهاتف، مع تثبيت الكاش والاختبارات قبل النشر.

## ما تغير

- نظام تصميم أخف وأكثر اتساقًا في `assets/styles/khadamati-v53.css`.
- صفحة بدء مختصرة، شريط سفلي أصغر، وبحث وطلب مباشر أقل ازدحامًا.
- بطاقات مزودين أكثر إحكامًا مع صور مهنية متخصصة ومهيأة للقص المربع.
- بوابة مزود متجاوبة في RTL وLTR، ومركز فرص لا يغطي الصفحة.
- محادثة كاملة الارتفاع مع Composer ثابت ودعم safe-area.
- لوحة إدارة تعرض الأولويات أولًا، ورسائل صحة نظام مفهومة بلا مسارات API خام.
- مدير Toast يمنع تراكم التنبيهات القديمة بعد التنقل.
- Cache جديد v53 مع مزامنة `index.html` و`public/index.html`.

## حماية البيانات القديمة

- لم يتغير مفتاح التخزين أو شكل السجلات الأساسية.
- الحقول الجديدة لها قيم افتراضية ضمن مسار التطبيع الحالي.
- صور v53 تطبق على الحسابات النموذجية فقط عندما تكون الصورة القديمة رسماً مولداً؛ الصور المرفوعة لا تُستبدل.
- لا توجد Migration تحذف حسابًا أو طلبًا أو محادثة أو اشتراكًا.

## متطلبات تشغيل خارجية

- GitHub Pages يشغل الواجهة وPWA فقط.
- الحسابات المشتركة وOTP والدفع وPush تحتاج خادم Render متاحًا، أسرار بيئة صحيحة، وقاعدة وتخزينًا دائمين.
- الدفع الآلي لا يعد ناجحًا قبل ربط مزود دفع والتحقق من Webhook على الخادم.
- إشعارات الهاتف الخارجية تحتاج VAPID وHTTPS وموافقة المستخدم؛ الإشعارات الداخلية تبقى متاحة.
