# خدماتي | Khadamati App

منصة عمانية لطلب الخدمات المحلية، مطابقة العملاء بالمزودين والشركات، وإدارة الطلبات والاشتراكات من الهاتف والكمبيوتر.

الواجهة الحالية هي واجهة `index.html` الأصلية الغنية: الصفحة الرئيسية، البحث، الأقسام، المزودون، بوابة المزود، لوحة الإدارة، المالية، الإعدادات، النسخ الاحتياطي، واتساب، والمفضلة. تمت إضافة مزامنة خادم لها حتى لا تبقى البيانات محصورة في جهاز المستخدم.

- واجهة عربية متجاوبة.
- خادم API باستخدام Python القياسي.
- قاعدة بيانات SQLite محفوظة افتراضياً في `khadamati.sqlite3` مع قراءة الملف القديم تلقائياً عند وجوده.
- حساب مستخدم مركزي برقم الهاتف ورمز اختياري وصورة وموقع محفوظ.
- طلبات مركزية بصور المشكلة والموقع والموعد والميزانية، مع مطابقة تلقائية للمزودين.
- قبول ذري للطلب يمنع قبول مزودين للطلب نفسه، وإشعارات موجهة للمستخدم والمزود والإدارة.
- خريطة Leaflet حية للمزودين والشركات مع الأقرب والتصفية حسب الخدمة والمنطقة.
- حفظ مركزي محمي لحالة الإدارة عبر `/api/classic-state`، ولا يقبل القراءة أو الكتابة دون صلاحية إدارة.
- لوحة إدارة بصلاحيات: مالك، مدير، دعم.
- تسجيل دخول منفصل للمزودين عبر رقم الهاتف والرمز السري.
- تسجيل مزودين مطور مع صورة شخصية، 5 صور أعمال، وملفات توثيق إلزامية وسرية للإدارة.
- إضافة وتعديل المزودين، قبول طلبات المزودين، وتعديل الإعدادات.
- رفع صور وملفات المزودين إلى مسار تخزين دائم عند النشر.
- تقييمات عملاء، شكاوى، متابعة جودة، ودرجة جودة لكل مزود.
- باقات واشتراكات ومدفوعات يدوية قابلة للتطوير لاحقًا للدفع الإلكتروني.
- تقارير UTF-8 وWord وصفحة طباعة PDF للإدارة.
- إدارة إعلانات مركزية تشمل المبلغ والمعلن والهاتف والمدة والتعديل والتمديد والأرشفة.
- استعادة رمز مؤقت، حذف حساب موثق، جلسات دائمة، وتسجيل محاولات الدخول الفاشلة.
- سجل عمليات للإدارة لتتبع التعديلات والقرارات.
- تسجيل طلبات التواصل وفتح واتساب/الاتصال داخل قاعدة البيانات.
- طبقة إشعارات WhatsApp Cloud API قابلة للتفعيل من متغيرات البيئة.
- نسخة احتياطية JSON من لوحة الإدارة.

## التشغيل المحلي

استخدم Python ثم افتح الرابط:

```powershell
python server.py
```

على Windows يمكن تشغيل الملف `start-khadamati-app.bat` لفتح الخادم والموقع تلقائياً.

الرابط:

```text
http://127.0.0.1:8080
```

اضبط رمز إدارة قوي قبل النشر العام:

```powershell
$env:KHADAMATI_ADMIN_CODE="ضع_رمز_قوي"
python server.py
```

## تفعيل WhatsApp Cloud API

اضبط المتغيرات التالية قبل التشغيل:

```powershell
$env:WHATSAPP_ACCESS_TOKEN="token"
$env:WHATSAPP_PHONE_NUMBER_ID="phone-number-id"
$env:WHATSAPP_API_VERSION="v20.0"
python server.py
```

بعدها استخدم تبويب `الإعدادات` في لوحة الإدارة لاختبار الإرسال. إذا لم تُضبط المتغيرات، يسجل النظام المحاولة في `whatsapp_logs` بدون إرسال فعلي.

## النشر

هذه النسخة مجهزة للنشر على Render عبر ملف `render.yaml` الموجود في جذر المشروع.

الإعدادات الجاهزة:

- `buildCommand`: `pip install -r requirements.txt`
- `startCommand`: `python server.py`
- `HOST=0.0.0.0` حتى يقبل الخادم زيارات الإنترنت.
- `KHADAMATI_DB_PATH=/var/data/khadamati.sqlite3` لحفظ قاعدة البيانات على قرص دائم.
- `KHADAMATI_UPLOAD_DIR=/var/data/uploads` لحفظ صور المزودين على قرص دائم.
- قرص دائم باسم `khadamati-data` على المسار `/var/data`.

خطوات Render:

1. افتح Render Dashboard.
2. اختر `New` ثم `Blueprint`.
3. اربط مستودع GitHub: `LLLX6/khadamati-app`.
4. عند طلب `KHADAMATI_ADMIN_CODE` ضع رمز إدارة قوي.
5. اضغط `Apply` وانتظر حتى يكتمل البناء.
6. سيعطيك Render رابطًا من نوع `https://khadamati.onrender.com`.

مهم: الخطة المجانية تعمل للتجربة، لكنها لا تضمن بقاء SQLite والصور بعد إعادة بناء الخدمة. التشغيل الإنتاجي يحتاج قرصاً دائماً أو PostgreSQL وتخزين ملفات خارجياً.

واجهة GitHub Pages تتصل افتراضياً بـ:

```text
https://khadamati.onrender.com
```

ولتغيير عنوان الخادم من المتصفح استخدم مفتاح `KHADAMATI_API_BASE` في `LocalStorage`.

لتفعيل واتساب الرسمي بعد النشر، أضف هذه المتغيرات من صفحة الخدمة في Render:

```text
WHATSAPP_ACCESS_TOKEN
WHATSAPP_PHONE_NUMBER_ID
WHATSAPP_API_VERSION=v20.0
```

ولتفعيل Push على الهواتف أضف مفاتيح VAPID:

```text
VAPID_PUBLIC_KEY
VAPID_PRIVATE_KEY
```
