# خدماتي | Khadamati App

منصة عمانية تربط العملاء بمزودي الخدمات والشركات، وتدير الطلبات والعروض والمحادثات والاشتراكات من واجهة عربية وإنجليزية مهيأة للهاتف أولاً.

## المكونات

- واجهة تطبيق ويب تقدمي في `index.html` و`public/index.html`.
- خادم Python في `server.py` وطبقة نطاق في `khadamati_domain.py`.
- SQLite افتراضياً مع مخطط PostgreSQL موثق للتوسع.
- Data Adapter يحافظ على توافق بيانات LocalStorage القديمة ويعزل الزائر عن جلسات المستخدم والمزود والإدارة.
- مطابقة دقيقة حسب القسم والخدمة والمنطقة والتوفر، على موجتين: أفضل 5 ثم التوسع إلى 10 عند الحاجة.
- محادثات وإشعارات مجمعة وطلبات وعروض وموافقات تواصل مرتبطة بالحساب والطلب الصحيحين.
- ساحة طلبات عامة آمنة، مع ترشيح مزود مطابق من مستخدم آخر؛ لا يُخطر المزود إلا عندما يختاره صاحب الطلب ويرسل الطلب إليه.
- سياسات خصوصية وجودة وشروط وإلغاء بالعربية والإنجليزية. البريد المركزي: `om.khadamati@gmail.com`.

## الباقات الإنتاجية

لا توجد إلا الخطط الخمس التالية، والعملـة `OMR`:

| المفتاح | المدة | السعر | أبرز الصلاحيات |
|---|---:|---:|---|
| `foundation_12m` | 12 شهراً | 0 | مرة واحدة للمزود الموثق، 3 خدمات، 5 ولايات، 30 استجابة شهرياً، وصول بعد 15 دقيقة |
| `basic_6m` | 6 أشهر | 6 | 5 خدمات، 10 ولايات، 60 استجابة، وصول بعد 10 دقائق |
| `basic_12m` | 12 شهراً | 10 | صلاحيات الأساسي نفسها |
| `professional_12m` | 12 شهراً | 20 | 10 خدمات، محافظتان، 25 ولاية، 150 استجابة، إشعار فوري وتقارير متقدمة |
| `business_12m` | 12 شهراً | 40 | 20 خدمة، 5 محافظات، 3 أعضاء، 3 فروع وصندوق مشترك |

الأسعار والصلاحيات والمدد قابلة للإدارة من الخادم. فترة السماح الافتراضية 14 يوماً، مع تنبيهات 30 و14 و7 و1 يوم ويوم الانتهاء. انتهاء الاشتراك يوقف الظهور واستقبال الطلبات ولا يحذف الحساب أو المحادثات.

## الخصوصية والأمان

- لا يظهر هاتف العميل للمزود إلا بعد موافقة مستقلة للقناة والطلب والمزود عبر `ContactConsentService`.
- الخطة المدفوعة لا تمنح حق الوصول إلى بيانات العملاء.
- وثائق التحقق خاصة بالإدارة المخولة.
- لا توجد كلمات مرور أو رمز إدارة ثابت في JavaScript، ولا تعرض الإدارة الرموز النصية.
- الأسعار الموثوقة والتحقق من الدفع وWebhook تتم في الخادم.
- OTP في الإنتاج لا يعمل كرمز تجريبي؛ يلزم ربط قناة إرسال حقيقية.
- كل تعديل إداري أو مالي حساس يسجل في Audit Log.

## التشغيل المحلي

```powershell
$env:KHADAMATI_ENV="development"
$env:KHADAMATI_DEV_ADMIN_CODE="ضع-رمزاً-محلياً"
$env:KHADAMATI_DEV_OTP_CODE="123456"
python server.py
```

أو شغّل `start-khadamati-app.bat` على Windows، ثم افتح:

```text
http://127.0.0.1:8080
```

للهاتف على شبكة Wi-Fi نفسها افتح `http://IP-الجهاز:8080`. قد يلزم السماح لـPython في جدار حماية Windows.

## متغيرات البيئة

استخدم `.env.example` مرجعاً. أهم القيم الإنتاجية:

- `KHADAMATI_ADMIN_CODE`: رمز إدارة قوي وغير محفوظ في المستودع.
- `KHADAMATI_OTP_PEPPER`: سر عشوائي لتجزئة رموز OTP.
- `KHADAMATI_MEDIA_SIGNING_KEY`: سر مستقل لتوقيع روابط الوثائق والوسائط الخاصة.
- `KHADAMATI_LOGIN_MAX_ATTEMPTS`, `KHADAMATI_LOGIN_LOCK_MINUTES`: حدود قفل محاولات الدخول.
- `KHADAMATI_MEDIA_URL_TTL_SECONDS`: عمر رابط الوسائط الخاصة الموقّع.
- `KHADAMATI_MAX_JSON_BYTES`: الحد الأعلى لجسم JSON قبل رفضه.
- `KHADAMATI_DB_PATH`, `KHADAMATI_UPLOAD_DIR`, `KHADAMATI_BACKUP_DIR`: مسارات تخزين دائمة.
- `KHADAMATI_PUBLIC_URL`: رابط GitHub Pages النهائي.
- `KHADAMATI_ALLOWED_ORIGINS`: الأصول المسموح بها.
- `KHADAMATI_SUPPORT_EMAIL`: بريد التواصل المركزي، وقيمته الافتراضية `om.khadamati@gmail.com`.
- `KHADAMATI_PAYMENT_*`: إعدادات Payment Adapter وWebhook.
- `WHATSAPP_*`: WhatsApp Cloud API الاختياري.
- `VAPID_*`: إشعارات الويب الاختيارية.

لا يصبح الدفع أو OTP إنتاجياً بمجرد إظهار الأزرار. يجب توفير مزود دفع/رسائل، أسرار صحيحة، HTTPS، واختبار Webhook الخادمي.

## Migration

قبل ترحيل الاشتراكات ينشئ الخادم نسخة SQLite احتياطية. الترحيل يعمل مرة واحدة بالمفتاح:

```text
KHADAMATI_SUBSCRIPTION_MIGRATION_V1
```

تُعطل الخطط القديمة بدلاً من حذف سجلاتها، وتُحفظ الاشتراكات والمدفوعات والتواريخ. لا تحذف قاعدة البيانات أو ملفات النسخ الاحتياطية أثناء الترحيل.

## الخريطة والتوفر وحدود الباقات

- الخريطة التفاعلية تستخدم Leaflet وOpenStreetMap، ولا تحتاج مفتاح API سرياً.
- لا تُنشأ إحداثيات افتراضية عند تعذر الموقع، ولا يظهر مزود على الخريطة من دون إحداثيات صحيحة وموافقة `mapVisible`.
- المزود العام في الخريطة والبحث يجب أن يكون معتمداً ونشطاً ومتاحاً وتسمح باقته بالظهور واستقبال الطلبات.
- المصدر الموحد لحدود الخدمات والأقسام والولايات هو كتالوج الباقات في الخادم. الواجهة تعرض الاستهلاك، لكن الخادم يعيد التحقق ويمنع التجاوز عبر API.
- عند خفض الباقة تُحفظ البيانات القديمة ولا تُحذف؛ يُمنع فقط إضافة عناصر جديدة حتى يصبح الملف ضمن الحد أو تتم الترقية.
- حالة توفر القسم أو الخدمة في الطلب المباشر تأتي من بيانات المزودين المؤهلين، ولا تسمح بمتابعة خدمة لا يوجد لها مزود متاح.

## النشر

`render.yaml` يجهز خادم Python. في Render أضف أسرار الإدارة وOTP والدفع يدوياً، واضبط قرصاً دائماً أو PostgreSQL وتخزين ملفات دائم قبل الإطلاق العام.

[نشر أو استعادة خادم خدماتي على Render](https://render.com/deploy?repo=https://github.com/LLLX6/Khadamati)

الخادم المنشور للتطبيق هو `https://khadamati-app-api.onrender.com`. يمكن فحصه من `https://khadamati-app-api.onrender.com/api/bootstrap`. إذا أعاد الخادم حالة `503` مع عبارة `Service Suspended`، فخدمة Render نفسها متوقفة وليست المشكلة في الواجهة أو رقم الهاتف. افتح الخدمة من لوحة Render واختر **Resume**، أو أنشئها من رابط النشر أعلاه. إذا كان الإيقاف مفروضاً من Render على الخطة المجانية، يلزم نقل الخدمة إلى خطة مدفوعة وفق سياسة Render.

GitHub Pages يقدم الواجهة الثابتة وService Worker فقط. لا يستطيع وحده تشغيل قاعدة البيانات أو تسجيل المزود أو OTP أو الدفع أو إرسال Push؛ هذه الوظائف تحتاج رابط خادم HTTPS متاحاً للواجهة.

رابط الواجهة المتوقع:

```text
https://lllx6.github.io/Khadamati/
```

## الاختبارات

```powershell
python -m py_compile server.py khadamati_domain.py
python -m unittest discover -s tests -p "test_*.py" -v
python tests/security-api.py
node tests/smoke-ui.js
```

لفحص أصغر هاتف والكمبيوتر غيّر `KHADAMATI_VIEWPORT_WIDTH` و`KHADAMATI_VIEWPORT_HEIGHT` قبل تشغيل اختبار الواجهة. اختبارات الإصدار الحالي معتمدة على 320×740 و390×844 و1280×900، وتشمل بقاء نافذة الدخول مفتوحة أثناء إدخال الرمز، حفظ الجلسة، والتحديث التلقائي للمحادثة.

اختبار الواجهة يبدأ خادماً ثابتاً معزولاً تلقائياً. لاختبار API شغّل خادم اختبار بقاعدة منفصلة في نافذة أولى:

```powershell
$env:KHADAMATI_ENV="test"
$env:KHADAMATI_ADMIN_CODE="رمز-اختبار-قوي"
$env:KHADAMATI_DB_PATH="$PWD\tests\__smoke.sqlite3"
$env:KHADAMATI_UPLOAD_DIR="$PWD\tests\__smoke-uploads"
python server.py
```

ثم في نافذة ثانية:

```powershell
$env:KHADAMATI_TEST_ADMIN_CODE="رمز-اختبار-قوي"
python tests/smoke-api.py
```

الاختبارات تغطي المستخدم والمزود والإدارة والطلب والمطابقة والمحادثة والموافقة والخرائط والهاتف، كما تتحقق من رفض التلاعب بمبلغ الاشتراك.
تفاصيل التهديدات والإصلاحات ونتائج الأدوات موثقة في `SECURITY_AND_QUALITY_AUDIT.md`.

## التوسع إلى PostgreSQL

راجع `POSTGRESQL_MIGRATION.md` و`postgres_schema.sql`. يمكن تصدير البيانات عبر:
