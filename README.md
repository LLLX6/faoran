# منصة خدماتي

نسخة مطورة من موقع خدماتي الأصلي الموجود في GitHub كمنصة تشغيل حقيقية.

الواجهة الحالية هي واجهة `index.html` الأصلية الغنية: الصفحة الرئيسية، البحث، الأقسام، المزودون، بوابة المزود، لوحة الإدارة، المالية، الإعدادات، النسخ الاحتياطي، واتساب، والمفضلة. تمت إضافة مزامنة خادم لها حتى لا تبقى البيانات محصورة في جهاز المستخدم.

- واجهة عربية متجاوبة.
- خادم API باستخدام Python القياسي.
- قاعدة بيانات SQLite محفوظة في `foran.sqlite3`.
- حفظ مركزي للواجهة الأصلية عبر `/api/classic-state`.
- لوحة إدارة بصلاحيات: مالك، مدير، دعم.
- تسجيل دخول منفصل للمزودين عبر رقم الهاتف والرمز السري.
- تسجيل مزودين مطور مع صورة شخصية، 5 صور أعمال، وملفات توثيق اختيارية.
- إضافة وتعديل المزودين، قبول طلبات المزودين، وتعديل الإعدادات.
- رفع صور وملفات المزودين إلى مسار تخزين دائم عند النشر.
- تقييمات عملاء، شكاوى، متابعة جودة، ودرجة جودة لكل مزود.
- باقات واشتراكات ومدفوعات يدوية قابلة للتطوير لاحقًا للدفع الإلكتروني.
- تقارير للإدارة عن الجودة، الإيرادات، الاشتراكات، وأفضل المزودين.
- سجل عمليات للإدارة لتتبع التعديلات والقرارات.
- تسجيل طلبات التواصل وفتح واتساب/الاتصال داخل قاعدة البيانات.
- طبقة إشعارات WhatsApp Cloud API قابلة للتفعيل من متغيرات البيئة.
- نسخة احتياطية JSON من لوحة الإدارة.

## التشغيل المحلي

استخدم Python ثم افتح الرابط:

```powershell
python server.py
```

الرابط:

```text
http://127.0.0.1:8080
```

رمز الإدارة الافتراضي:

```text
1995
```

حسابات المزودين التجريبية تستخدم آخر 4 أرقام من الهاتف كرمز دخول:

```text
سالم البلوشي: 91234567 / 4567
النخبة للتنظيف: 92345678 / 5678
```

لتغيير الرمز في الإنتاج:

```powershell
$env:FORAN_ADMIN_CODE="ضع_رمز_قوي"
python server.py
```

## تفعيل WhatsApp Cloud API

اضبط المتغيرات التالية قبل التشغيل:

```powershell
$env:WHATSAPP_ACCESS_TOKEN="token"
$env:WHATSAPP_PHONE_NUMBER_ID="phone-number-id"
$env:WHATSAPP_API_VERSION="v20.0"
python server.py
```

بعدها استخدم تبويب `الإعدادات` في لوحة الإدارة لاختبار الإرسال. إذا لم تُضبط المتغيرات، يسجل النظام المحاولة في `whatsapp_logs` بدون إرسال فعلي.

## النشر

هذه النسخة مجهزة للنشر على Render عبر ملف `render.yaml` الموجود في جذر المشروع.

الإعدادات الجاهزة:

- `buildCommand`: `pip install -r requirements.txt`
- `startCommand`: `python server.py`
- `HOST=0.0.0.0` حتى يقبل الخادم زيارات الإنترنت.
- `FORAN_DB_PATH=/var/data/foran.sqlite3` لحفظ قاعدة البيانات على قرص دائم.
- `FORAN_UPLOAD_DIR=/var/data/uploads` لحفظ صور المزودين على قرص دائم.
- قرص دائم باسم `faoran-data` على المسار `/var/data`.

خطوات Render:

1. افتح Render Dashboard.
2. اختر `New` ثم `Blueprint`.
3. اربط مستودع GitHub: `LLLX6/faoran`.
4. عند طلب `FORAN_ADMIN_CODE` ضع رمز إدارة قوي.
5. اضغط `Apply` وانتظر حتى يكتمل البناء.
6. سيعطيك Render رابطًا من نوع `https://faoran.onrender.com`.

مهم: القرص الدائم ضروري حتى لا تضيع قاعدة البيانات والصور بعد إعادة التشغيل. إذا أردت تجربة مجانية بدون قرص دائم يمكن تشغيل الخدمة بدون `disk`، لكنها لا تصلح كتشغيل إنتاجي يحفظ بيانات العملاء.

لتفعيل واتساب الرسمي بعد النشر، أضف هذه المتغيرات من صفحة الخدمة في Render:

```text
WHATSAPP_ACCESS_TOKEN
WHATSAPP_PHONE_NUMBER_ID
WHATSAPP_API_VERSION=v20.0
```
